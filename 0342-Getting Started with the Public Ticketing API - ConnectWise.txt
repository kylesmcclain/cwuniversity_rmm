Overview

Below are prerequisites and other ticketing API basics to review before configuring your API operations.

Prerequisites
Must have a Super Admin account in the Asio™ Platform (ITSupport Portal).
Enable the PSA Integration option per the PSA system used. PSA Integration enablement options are available in the Asio™ Platform (ITSupport Portal) by clicking on Settings and navigating to Account Administration > Integration > PSA Integration.
Authentication

To enable ConnectWise to properly and safely authenticate API calls, ensure the Integrator User and Callback details are up-to-date in the PSA Integration page (under Settings > Account Administration > Integration > PSA Integration).

Callback Mechanism

A callback is a programming practice of transmitting data to another program. A ConnectWise ticketing API callback occurs when any create or update action is dispatched on a ticket object in ConnectWise.

Use the Callback URL field in the PSA Integration page to specify a standard or custom callback URL. When the callback in RMM is triggered, the Ticket ID and the Action will be added to the URL by GET/Post method, as follows:

action = <new/update>

NOCTicketID = <numeric value of NOC Ticket ID>

NOCTicketNotesID = <numeric value of NOC Ticket Notes ID, if added in Ticket Object>

Notes 

CW RMM will only send a callback on the URL specified to send a request. The PSA System should query the data changed or modified in CW RMM.

If you are using Microsoft visual studio for generating the WSDL, we recommend you generate a client proxy class using the svcutil.exe tool included with the .Net framework. Below is the sample command to generate the proxy class. Once you execute the command, you will see two files in the current directory path (such as TicketService.cs and output.config). Add these two files into your project and use them to make the API call.

E:\> svcutil https://services.itsupport247.net/ti...rvice.svc?wsdl /namespace:*,YourNamespace /serializer:xmlserializer

Note: Replace “YourNamespace” in the command above with the actual name you want to use.

Ticket Source

ConnectWise RMM tickets are generated in the following ways:

Automatically by RMM agent
By ConnectWise NOC or Help Desk Service technicians
By MSPs via public ticketing API
By MSPs directly from Asio™ Platform (ITSupport Portal)
Ticket Types

ConnectWise RMM tickets can be:

Resource-specific
Site-specific
Adhoc, created by the MSP and not necessarily affiliated with any ConnectWise RMM site or resource
