Overview

During the public ticketing API integration process, the User Acceptance Test below is initially reviewed during PSA Qualification and Configuration (Phase 1) and then executed during PSA Authorization (Phase 3) to ensure that the integration was successfully completed based on the API specifications.

Note: Professional Services Automation (PSA) references also apply to MSPs who synchronize their own ticketing system with ConnectWise.

Prerequisites
PSA has mapped ticket fields, such as priority and status, based on ConnectWise’s supported options.
A mapping is available to identify the end-client based on ConnectWise's sites.
MSP has configured the integrator user name, password, and callback URL in the Asio™ Platform (ITSupport Portal).
RMM agents are installed on the MSP's client machines and are running successfully.
User Acceptance Test
  Test Case	  Process for MSP and/or ConnectWise
Positive Scenarios
	Retrieve newConnectWise ticket	
Automated ticket is generated by ConnectWise's alert processing infrastructure, such as when a server goes down.
ConnectWise system invokes the MSP's callback URL with predefined format and fields.
MSP uses ConnectWise's "GetTicket" API operation to retrieve the ticket details.
MSP chooses to have that ticket appear in their PSA portal, in an email, or other medium.

	Retrieve updated ConnectWise ticket	
NOC updates a ticket status to Resolved.
ConnectWise system invokes the MSP's callback URL with predefined format and fields.
MSP uses ConnectWise's GetTicket API operation to retrieve the ticket details.
MSP decides how to close the ticket and communicates to clients.

	Update ticket owner	
MSP updates the owner of a ticket using their PSA portal.
MSP uses ConnectWise's UpdateTicket API operation to send the updates.
Ticket is updated in NOC portal.

	Create ticket	
MSP creates a ticket from PSA portal and assigns it to the NOC.
MSP uses ConnectWise's CreateTicket API operation to create a ticket in ConnectWise system.
Ticket is added to NOC portal.

	Create ticket in Asio™ Platform (ITSupport Portal)	
MSP creates a ticket from the Asio™ Platform (ITSupport Portal) and assigns it to NOC.
ConnectWise system invokes the MSP's callback URL with predefined format and fields.
MSP uses ConnectWise's GetTicket API operation to retrieve the ticket details.
MSP chooses to have the ticket appear in their PSA portal.

	Add note to ticket	
MSP adds a note to an existing ticket using their PSA portal.
MSP uses ConnectWise's UpdateTicket API operation to send the updates.
Notes are added to the ticket in NOC portal.

	Retrieve ticket notes	
ConnectWise's Help Desk Service adds a note in a ticket.
ConnectWise system invokes the MSP's callback URL with predefined format and fields.
MSP uses ConnectWise's GetTicket API operation to retrieve the updates.
MSP chooses to have the ticket note appear in their PSA portal.

	Close ticket	
MSP closes an existing ticket from their PSA portal.
MSP uses ConnectWise's UpdateTicket API operation to close the ticket
Ticket is closed in NOC portal.

	Retrieve all open tickets	
MSP activates the ticketing integration from the Asio™ Platform (ITSupport Portal).
ConnectWise system invokes the MSP's callback URL with predefined format and fields for all open tickets.
MSP uses ConnectWise's GetTicket API operation to retrieve the ticket details.
All open ConnectWise tickets are synced to PSA portal.

Negative Scenarios
	Unsupported character excluded during sync	
A character not supported by RMM is added to a ticket note in PSA portal. (ConnectWise only supports ASCII characters Dec 32 to Dec 126.)

Regular expression : [^A-Za-z0-9_@\[\]\{\}|""\\#^<>`~,;?\'=/*\&$%!\+\(\)\s\t\r\n\:\.-]

Example: MSP adds ° in the ticket description.

MSP uses ConnectWise's UpdateTicket API operation to update the ticket with an unsupported character .
The character is excluded in the NOC portal but all other text is displayed.

	Unsupported status change produces error	
MSP changes the ticket status from Closed to Open in PSA portal. (RMM does not allow this.)
MSP uses ConnectWise's UpdateTicket API operation to update the ticket with the unsupported status change.
MSP receives an error.

	Unrelated ticket created in PSA causes inefficiency in Asio™ Platform (ITSupport Portal)	
MSP creates a ticket in PSA portal that is unrelated to ConnectWise's product suite, such as a ticket related to a VOIP service. (MSP's should ensure to use ConnectWise's API for only ConnectWise products and services.)
The un-workable ticket appears in the NOC portal and causes unwanted downtime for the NOC and the MSP.
