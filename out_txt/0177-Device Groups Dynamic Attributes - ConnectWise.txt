Hardware	Chassis Type	

Specifies the type of chassis for the device.

Note: For chassis types below, some additional information when value is -

Other: This code is used when the chassis type does not fit into any of the predefined categories provided by the SMBIOS specification. It indicates that the chassis type is known but does not match any of the standard types listed.
Unknown: This code is used when the chassis type is not known. It indicates that the system is unable to determine the type of chassis.
	STRING
Subresource Type	Indicates whether the device is physical or virtual.	STRING

Note:

Sometimes, no information is available related to TPM chip, in this case the device will neither fall under enabled = true or enabled = false.
Applies to windows endpoints only.
	BOOLEAN
