Overview

Custom Field Monitors allow you to define monitors for the custom fields created in the system. Before creating a monitor, you need to add a custom field in the system. Please review the Custom Fields documentation for the process on how to create these.

How to create a Custom field Monitor

Navigate to  Endpoints > Alerts > Monitors.

Click Add Monitor.

Provide a Name for the new monitor. Optionally you can provide a Description of your new monitor.

Select the Type as Custom field from the drop-down for creating the appropriate monitor.

Select the Family and Severity from the drop-down menus.

To enable the Mapper functionality, you need to turn on the toggle. To learn more about the Mapper functionality review this documentation.

Select the Device Availability from the Metric type drop-down
Under the Condition section:
Custom Field Type - This allows you to select the custom field types which can be used to trigger alert when the value changes in them. 
Datepicker - For fields where values will be Date and time stamps.
Equals - Matches with the value.
Not Equal - Does not matches with the value.
Greater Than - The value is greater than.
Less Than - The value is less than.
DropDown: - For fields where values will be one of the available options.
Contains - The value must contain the keywords.
Does not contain - The value does not contains the keywords.
Equals - Matches with the value.
Not Equals - Does not matches with the value.
Flag - For fields where values will be Boolean (Yes or No).
Equals: - Matches with the value.
Not Equals: - Does not matches with the value.
Text box: - For fields where values will be textual.
Contains: - The value must contain the keywords.
Does not contain: - The value does not contains the keywords
Equals: - Matches with the value.
Not Equals: - Does not matches with the value

Custom Field Name: - Based on the custom-field-type selection, it will display the lists of custom-fields which are pre-created with custom field type option.

 

If no custom fields are created, it will not display any details.

You can also select a custom Automation Task to be executed when the condition is met on the endpoints.

This section allows you to select the target endpoint on which the alerts should be triggered.
