Overview

Every Linux device must have the agent installed in order for it to report back to the ConnectWise Asio® platform with monitoring information. The Linux Agents can be downloaded from the Asio platform and installed on your device.

Before installing a server agent, you need to create user accounts so that critical alert notifications can be defined in Escalation Templates, and then create the site. For details, see Setting Up Your Asio platform. Once your site has been set up, please check the agent prerequisites and compatibility information available in Prerequisites.

Our Linux agent support has been expanded to include Redhat and Fedora, and 64-bit versions. Support has also been added for standard proxy types like HTTP, HTTP1, SOCKS4, SOCKS4A and SOCKS5.

Linux Agent Requirements
Servers only 64-bit with kernel version 2.6 and above
Hardware Requirement
CPU: Pentium 4 or comparable processor.
RAM: 2 GB and above.
Disk Storage: 1.5 GB on the drive where the agent will be installed.

Disk storage for auto-updating agents: An additional 2 GB of storage should always be available on the machine for the agent to get auto-updated.

Note: Not meeting the above requirements may result in failure of agent installation/auto-update.

Software Requirement
Install the curl library (apt-get install libcurl4).
Install bzip2 (apt-get install bzip2) or  zip (apt-get install zip)
Remote assistance is integrated with ScreenConnect, which requires OpenJDK 9 or above. Please reference System Requirements.
GLIBC 2.14 or above (Run the command to check GLIBC version “sudo ldd –-version”)

Note: The Linux operating systems mentioned above support the following proxy types: HTTP, HTTP1, SOCKS4, SOCKS4A and SOCKS5.
