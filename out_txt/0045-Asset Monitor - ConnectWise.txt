Overview

Asset monitors allow you to set monitoring for the values of attributes from a target endpoint.

Navigate to Endpoints > Alerts > Monitors to create an asset monitor.

Create an Asset Monitor

Navigate to  Endpoints > Alerts > Monitors.
Click Create Monitor.
Provide a Name for the new monitor. Optionally, you can provide a Description of your new monitor.
Select the Type as Asset from the drop-down for creating the appropriate monitor.
Select the Family and Severity from the drop-down menus.
To enable the Mapper functionality, you need to turn on the toggle. To learn more about the Mapper functionality review this documentation.
Click Select Targets to create a specified endpoint monitor. Sort by Companies, Sites, Device Groups, Devices, or All Resources. You can select specific machines in the sites using a Device Type filter. The machine name and friendly name will automatically be fetched from device.
Complete the required fields in the Conditions section:

Category - This allows you to select the supported asset categories for which monitors will be created.
Parameter - You can select parameters that belong to the asset category that is selected.
Keywords - This allows you to specify the values that can be used to trigger the alerts.
(Optional) Click Add Automation to select a custom Automation Task to be executed when the condition is met on the endpoints.
Complete the required fields in the Ticket Resolution section to determine the conditions for automatic ticket resolution:

Category - This allows you to select the supported asset categories for which monitors will be created.
Parameter - You can select parameters that belong to the asset category that is selected.
Keywords - This allows you to specify the values that can be used to trigger the alerts.
(Optional) Select a Monitor Output. 

Click Save to create and apply the monitor to the targets.

Note: Monitors are triggered based on the configuration set/defined.

Conditions

Condition options display based on the parameter selected. Conditions include:

Contains - The value must contain the keywords.
Does not contain - The value does not contain the keywords.
Greater Than - The value is greater than.
Less Than - The value is less than.
Equals - Matches with the value.
Not Equal - Does not match the value.
Logical Operators - Logical operators such as AND / OR can also be used for the conditional parameters, where it is supported.
