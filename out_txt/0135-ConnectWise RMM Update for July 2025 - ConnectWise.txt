July 30, 2025
Agent

Endpoints with RMM Agent installed will receive an Agent update version 4.0.3. The rollout will start from 30 July in a phased manner. The new Agent setup version 4.0.3 will be enabled from 31 July in a phased manner for all regions.

Note: Always use the latest version while installing the Platform Agent to access new functionalities and ensure that it has the most up-to-date features. Make sure to update the GPO, MS Intune, or any other method used to install the Agent.

Issues Addressed
Known Issue	Description
 	Agent heartbeat is fetched via a secure URL, so agents previously shown offline incorrectly will now appear online on the Asio portal.
 	Fixed an issue where an empty configuration file on the machine was preventing the RMM agent from auto-updating.
 	Fixed an issue where active applications were getting terminated or restarted on macOS when RMM Agent was being uninstalled.
ISSUE-7514	Fixed an issue where the Event Log Plugin was causing high CPU utilization when the event log size was large.
July 24, 2025
Automation
Enhanced the Script Editor by migrating all script functions from modal dialogs to peek panels for a more streamlined and consistent user experience.
Issues Addressed
Known Issue	Description
ISSUE-8114	Fixed an issue where the task execution details were not appearing in the Automation Execution Summary and the Detail Reports.
 	Fixed an issue where the script parameter inputs were limited to 100 characters; the limit has now been increased to 1000 characters.
 	Fixed an issue where the task parameters could be manipulated in a way that allowed unintended code execution under specific conditions.
July 15, 2025
ScreenConnect
The ScreenConnect View feature is now available in the Asio Top Navigation pane as Remote View. Review this Remote View documentation.
July 11, 2025
Uninstall Agents API
API to uninstall agents is now available via OpenAPI.
Issues Addressed
Known Issue	Description
PCH-5722	Addressed issue where feature updates did not deploy when the endpoint comes online in the middle of the Schedule window.
PCH-5878	Feature Update Deployment was getting marked as installation completed, preventing the finalization process.
PCH-5841	Superseded or Patches that are no longer missing on the endpoint will not be marked as success anymore.

Alert Name: Microsoft Hyper V - Stale AVHD Files
We improved the logic for the Under Research alert "SQL Server Percent Log".
We improved the logic for the condition - “Security Notification - A user account is added to Security group”.
