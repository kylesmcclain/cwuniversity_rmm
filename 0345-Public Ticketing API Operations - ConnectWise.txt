Overview

Below are the operations supported in ConnectWise's public ticketing API.

GetTicket:
<soap:Envelope xmlns:soap="http://www.w3.org/2003/05/soap-envelope" xmlns:v1="http://www.continuum.net/its/v1_0">
	<soap:Header xmlns:wsa="http://www.w3.org/2005/08/addressing" >
		<UserID xmlns="TicketService"></UserID>
		<Password xmlns="TicketService"></Password>
		<TokenID xmlns="TicketService"></TokenID>
		<wsa:Action>GetTicket</wsa:Action>
		<wsa:MessageID>uuid:ecf1afb0-176a-4e55-b28e-e0773bce7dab</wsa:MessageID>
		<wsa:To>https://services.itsupport247.net/ticketing/1.0/TicketService.svc/soap</wsa:To>
	</soap:Header>
	<soap:Body>
		<v1:GetTicket>
			<v1:request>
				<v1:Id></v1:Id>
			</v1:request>
		</v1:GetTicket>
	</soap:Body>
</soap:Envelope>

CreateTicket:

Creates a ticket in the ConnectWise system

Notes

If a valid contact (i.e. contact is active and belongs to the same site) is passed then the contact will be associated/linked while creating the ticket.

If an invalid contact id is passed or the contact is inactive, then the system will throw an error.

UpdateTicket:

Updates an existing ticket in ConnectWise system

Notes

If a valid contact (i.e. contact is active and belongs to the same site) is passed then the contact will be associated/linked with the existing ticket.

If the ticket is already linked to a contact, then the system will unlink the previous contact and link with a new contact id

If an invalid contact id is passed or the contact is inactive, then the system will throw an error.

If passed zero ("0") value into the contact id or no contact id passed and the existing ticket has already a contact associated, then the system will de-associate/unlink the contact from the existing ticket.

		<wsa:MessageID>uuid:ecf1afb0-176a-4e55-b28e-e0773bce7dab</wsa:MessageID>

	</soap:Header>

GetTicketStatuses:  

Gets a list of all ticket statuses from the ConnectWise system.

Status

	

Status Details




Acknowledged

	

This status would be used by RMM when we would acknowledge the ticket being received on our end. Once the ticket has been acknowledged it would be assigned to a tech to work on the same.




Approval Needed

	

This status would be used by RMM when we would need some approval from Partner to continue working on the issue further.




Approved

	

This is a partner status and it would be used by a partner when they are providing some update on a ticket or reverting with approval to work on the issue.




Closed

	

This status would be used by partner and NOC both. Ticket once closed in RMM cannot be reopened.




Info needed / Work Assigned

	

This status would be used by NOC when we would need additional information from your team to continue working on a issue.




Interim Notification

	

This status would be used by RMM when we are continuing to work on a issue but wanted to provide you with a interim status of the work performed and next action plan.




New

	

This status would be used by us and partner. This notifies a new ticket either on either systems.




NOC Completed

	

This status would be used by RMM when we would be updating the ticket once the work requested on our end has been completed. This doesn't correspond to Complete as the partner can still update on the ticket that they feel some work was missed or you would like to request additional changes for the issue.




Not Approved/ Further Info Needed

	

This will be used by the partner when they do NOT approve for work to be done by ConnectWise team or he needs more information from us about the issue.




Pending

	

This status is used by the partner when he is awaiting some information on their end before we can commence the work further.




Schedule

	

This status would be used by NOC when we would be scheduling a ticket to be worked at a specific interval or timeframe.




Task Missed/Not Completed

	

This status will be selected when the ConnectWise team was unable to complete a ticket.




Waiting Customer

	

This status would be used by the partner when they are either waiting on some third party or end client to get confirmation for NOC to work on the issue further.




Work In Progress

	

This status would be used by NOC when we would start working on a specific issue.

Note that the following Status id numbers are internal for partners and are not used by the Asio System:

Status id: 205
Status id: 206
Status id: 207
