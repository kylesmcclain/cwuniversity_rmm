Overview

Advanced Network Monitoring is an add-on product for RMM Pro and Premium packages, enabled at the site level. For an additional cost, you can upgrade your embedded Network Monitoring capabilities including:

Configuration backup comparison and history so you no longer need to guess what changed between configurations. Instantly bring back any previous configuration from the version history when needed (requires SSH). 

60 out-of-the-box SNMP alerts to leverage our best practices recommendations and stay on top of network events (requires SNMP). 

Real-time statistics for better tracking of bandwidth, utilization, and packets. 

NorthStar troubleshooting tool that maps the path of a device to the internet (requires SNMP). 

Important: Billing is not prorated. No matter when the feature is enabled during the monthly billing cycle, you are billed for the entire month. The option must be disabled before the first day of the month to avoid being billed for the following month. Even if the feature is disabled mid-month, you still have access to use the feature for the remainder of the month. 

Activating Advanced Network Monitoring
Enable the Network Management Upgrade in the Marketplace
In the Asio platform, navigate to Integrations > My Integrations.
Locate the Networking Management Upgrade tile, click the three dots icon, and select Activate.

On the next screen, click Integrate.

The consent form will open. This is to remind you that this is paid upgrade. From the drop-down, select I agree and then click Save and Proceed.

Click Complete Setup.

Upgrade your Network Monitoring Policy

You must assign a new policy to sites that you want to use Advanced Network Monitoring on.

In the Asio platform, navigate to Endpoints > Policies.
Click Add Policy.
For the Category field, select Sites.
For the Type, select Network Monitoring.
Name your policy.

In the Site Type drop-down, select Advanced Network Monitoring.
Add your new policy to a package and assign it to the sites you want to upgrade. For information on how to do this, refer to the Packages Documentation.
Downgrade to Basic Network Monitoring

Important: You must change the policy back to Basic Network Monitoring, or remove the policy from a site or you will continue to be billed for Advanced Network Monitoring. Removing the integration from the ConnectWise Marketplace does not disable Advanced Network Monitoring.

We have three options to downgrade your Network Monitoring.

Downgrade the Policy Option
In the Asio platform, navigate to Endpoints > Policies.
Locate the policy with Advanced Network Monitoring and select it.
Under Site Type, select Basic Network Monitoring.

Click Save.
Remove the Policy from a Package / Site Option

If you want to keep the policy but remove it from the package, choose this option.

In the Asio platform, navigate to Endpoints > Packages.
Locate the package that contains the Network Monitoring Policy and select it.
Remove the Network Monitoring Policy from the package by dragging it back to the Sites section of Available policies, or click the delete icon.

Remove the Package from a Site
In the Asio platform, navigate to Endpoints > Packages > Sites tab.
Locate the site you want to remove the package from.
Select the site and click Remove Assignment.

Note: If you want to completely remove Advanced Network Monitoring from your instance, follow the steps in the Downgrade the Policy Option, and then turn off the integration from the Marketplace page.

Deploying the Network Probe

If you have not yet deployed a probe on your Advanced Network Monitoring sites, use the following steps to make sure that you can use Advanced Network Montioring.

The first step to begin discovering devices is to designate at least one of your Site's RMM agents as the Network Probe.

The Network probe is a service that uses several protocols to gather information about your network, such as topology details, configurations, and network statistics. The Network probe is configured to be associated with the Company and Site where the RMM agent is installed.

When an RMM device is designated as a Network Probe, the Company and Site associated with that RMM agent are automatically created within Auvik.

Steps to deploy a Network Probe
Navigate to the Asio Platform > Endpoints > Devices.
On the Devices page, select the Windows device that you want to be designated as the Network Probe for the site.
Click on Manage > Network Probe.

On the confirmation screen, click Confirm.

You will receive a message when the Network Probe deployment is successfully initiated.

The Network Probe deployment takes around ten minutes to complete on the endpoint.

RMM Devices where the Network Probe is successfully installed and running shows under the Device Type.
Multiple Probe Deployment

A site can have multiple Network Probes to maximize coverage of all Networks and Devices. However, you can only create one Network Probe at a time. For sites where the Network Probe deployment is initiated for the first time, immediately adding another collector will fail as the Company or Site registration within Auvik is in progress. After this process is finished you can create another. 

If you have multiple probes at a site, you will need to complete the following process:

In the Asio Portal, navigate to Endpoints > Network. 
In the top right corner of the page, click the Manage button.

In the Auvik Portal, navigate to Admin > Auvik Collectors.
Search for and select the probes you have created.

Then click APPROVE.

Uninstalling a Network Probe 

Note: When you remove a probe from a site in Asio, the probe entry will not be removed in the Auvik portal. This probe will show as disconnected, but will no longer be collecting any data. 

To uninstall a network probe, navigate to the Asio platform > Endpoints > Devices.
On the Devices page, select the checkbox for the device that you want to remove the Network Probe from.
Click Manage > Network Probe.

In the pop-up, deselect the Enable Network Probe checkbox.
Click Confirm.

Network Device Credentials 
Navigate to Endpoints > Network and sign in. You must navigate to the Network page from Asio to authenticate; attempts to authenticate outside of the Asio platform may result in an error. Device credentials are required to get details from the Network Devices.
A prompt appears within the Auvik UI to add SNMP credentials when SNMP-supported devices are discovered. Click Manage Credentials in the pop-up.
In Auvik, you can directly add credentials to a Partner, Company, or Site level from Admin > Discovery > Manage credentials. 
Company and Site Provisioning 

You can start using the solution after you complete the Account registration. The provisioning and mapping of ConnectWise Companies (Called Multi-site in Auvik) and Connectwise Sites (Called Sites in Auvik) occur automatically.

The registration of the Companies and Sites is triggered when the User designates machines as Network Probes on the Site. More details on Network Probe deployment can be found in the Network Management section.

User Provisioning 

Important: Any user given permission to the Network Devices page can log in to Auvik and see all of their Companies and Sites. We do not recommend giving Network Devices permission to a user with Limited site access.

Network Device permissions are not enabled by default for Manager or Technician default roles. In ConnectWise Home, under User & Application Settings > Application Roles, you can manage permissions for Managers or Technicians at the user or role level. For more information on adding custom permissions to roles, refer to Application Roles.

When a ConnectWise RMM user is added to Asio and assigned a role with permission to Network Devices, that user is also added to the network monitoring solution. 

Primary Super Admin / Super Admin (by Default)
Or, permission for Devices - Network Devices added to an assigned role

When a user is granted permission to access Devices - Network Devices, they see the Network available via Endpoints > Network in Asio. Each role within the Asio platform corresponds to a custom role in Auvik with pre-defined permissions.

For example:

The Super Admin or Manager role can modify Alert Threshold within Auvik
The Technician role can only view the values of the Alert Threshold within Auvik

Users can also be viewed in the Auvik UI. However, users, roles, and authorizations can only be managed in ConnectWise Home.

CW RMM Users are authenticated to the Auvik UI when they click on a Device on the Network Devices page. Users can only access Auvik from the Asio platform via Endpoints > Network. The authentication only works from the Asio platform; direct access to the Auvik Portal will fail.
