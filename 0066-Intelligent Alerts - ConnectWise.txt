Overview

ConnectWise RMM™ intelligent alerts are a plug-and-play monitoring system that automatically applies predefined conditions based on the endpoint type (server, desktop, etc.). 

When the ConnectWise RMM agent is installed on an endpoint, the agent performs an automatic scan to identify active services, open ports, and running processes. Based on this scan and the RMM product package in use (Core, Standard, or Premium), the system applies a tailored set of intelligent alerts. 

Please refer to Intelligent Alerts Conditions for more detailed information about each condition.

Manage Intelligent Alerts

The Intelligent Alerts page provides centralized visibility and control over monitoring conditions across endpoints, companies, and sites. You can access this feature from multiple areas of the platform, each offering a different scope of visibility.

Note: The displayed alert data is context-sensitive based on the navigation path. However, automations, mappings, and monitor output settings are global—changes made in Edit Intelligent Alert apply across all applicable endpoints and devices.

Scope

	

Path




All endpoint monitoring

	

Endpoints > Alerts > Intelligent Alerts




Company and site monitoring

	

Clients > Companies > [select a company] > Monitoring > Intelligent Alerts




Endpoint monitoring

	

Endpoints > Devices > [select a device] > Monitoring > Intelligent Alerts

All Endpoint Monitoring Conditions

Note: The monitoring conditions available on this page are determined by your ConnectWise RMM product package (Core, Standard, Premium).

Navigate to Endpoints > Alerts > Intelligent Alerts to access this view.

This tab provides a unified list of all preconfigured intelligent alert conditions. It serves as the central hub for managing alert logic, routing, and automation. To learn more, refer to the Descriptions and Configuration sections below.

Intelligent alert conditions are auto-applied based on endpoint type (e.g., server, desktop, VM). 
Click the condition name to customize alert actions, such as automation or ticket generation.
Changes made in Edit Intelligent Alert apply across all applicable endpoints and devices.

Download a Visualization
Navigate to Endpoints > Alerts > Intelligent Alerts.
Click the kebab icon.

Select your preferred file type from the drop-down.
Company and Site Monitoring

After the agent is installed for a device associated with an Asio company, you can navigate to the company detail page and manage the intelligent alert monitoring conditions for all endpoints within the company. To learn more, refer to Intelligent Alert Configuration.

Navigate to Clients > Companies > [select a company] > Monitoring > Intelligent Alerts

To manage at the site level, first select a company and then select the site. From the site detail page, click Monitoring > Intelligent Alerts.
Click the condition name to customize alert actions, such as automation or ticket generation.
Changes made in Edit Intelligent Alert apply across all applicable endpoints and devices.
From this view, you can also manage Suspension and Custom Monitors for the company level.

Endpoint Monitoring Conditions

After the agent is installed on a device, you can access a detailed view of intelligent alerts specific to that endpoint. To learn more, refer to the Descriptions and Configuration sections below.

Navigate to Endpoints > Devices > Device Detail > Monitoring > Intelligent Alerts to view this page.

This page provides a highly targeted view of alerting behavior and automation for the selected endpoint. Technicians and users with access to Intelligent Alerts can:

Review active alerts assigned to the endpoint. 
Customize alert behavior for the endpoint.
Assign automation tasks to streamline remediation.
Suspend alerts during maintenance or exception periods.

Configuration and Output

Each alert condition is pre-configured based on endpoint type. You can set up automated tasks within the Edit Intelligent Alert window. Changes made in Edit Intelligent Alert apply to all applicable endpoints and devices.

Automation Tasks

Follow the steps below to configure an automation task for an intelligent alert:

Navigate to Endpoints > Alerts > Intelligent Alerts.
Click the condition name in the Intelligent Alert list view.

In the Automation pod, click Add Automation.

Search for and select the task you want to enable for this condition. If you do not have an automation for this condition yet, you can create one in Automation > Tasks and then return here to select it. Please note the following about automations:
The list of available automations is populated based on your permissions.
Users with permissions to create automation tasks can save the automation for the monitoring condition.
The system displays automation tasks that do not require parameters. Please refer to these out-of-the-box (OOTB) automation tasks for parameterized scripts. All custom scripts/tasks created are available for selection.

Click Select. 
Click Select Targets to identify the specific resources this automation applies to.

Sort endpoints by Companies, Sites, Device Groups, Devices, or All Resources using the radio buttons at the top. 
Use checkboxes to select endpoints. 
Tip: Selecting from Device Groups can help narrow down device types and preferences. This task applies to all future devices added to the device group.
Click Save Selection.
Click Save to finish editing the alert configuration.

Intelligent Alert Mapper

Note: This feature is applicable only for partners using ConnectWise PSA with Policies.

The mapper functionality allows you to associate each Intelligent Alert—whether configurable or non-configurable—with specific attributes such as Service Board, Type, Sub-Type, and Item. This mapping enables differentiated handling of alerts based on monitoring conditions.

Enable the Mapper

Note: This functionality requires Alert Management - Intelligent Alert > Mapper permissions. Refer to the Intelligent Alerts permission comparison for more information.

By default, the Mapper toggle is turned off. To activate it:

Switch the Mapper toggle on. 
Select the values from the drop-down. These selections override the PSA policy configuration. 

Mapper Field Descriptions

This feature is applicable only for partners using ConnectWise PSA with Policies.

Field

	

Description




Service Board

	

Set the service board the ticket is routed to upon creation.




Type, Sub-Type, Item

	

Type → Sub-Type → Item: These fields are interdependent.

Selecting a Type filters the available Sub-Types.
Selecting a Sub-Type filters the available Items.
Example: Choosing Tier 1 as the Type shows Sub-Types associated with Tier 1 only.



Priority

	

Override the system's default ticket priority.




Creation Status

	

Override the system's default creation status.




Closure Status

	

Select one of the predefined statuses created by the RMM-PSA integration.

Please note that only statuses marked as Closed appear in the drop-down menu.

Monitor Output

The Monitor Output functionality lets you control ticket generation behavior for intelligent alerts. 

To select your ticket preference:

Navigate to Endpoints > Alerts > Intelligent Alerts.
Click on the Alert condition from the list that you want to turn ticket generation on/off for.
In the Monitor Output section, click the Output drop-down.
Select from the following options:
Generate Ticket: A ticket is created immediately when the alert condition is met.
Generate Ticket with Delay: A ticket is created after a specified delay (5-60 minutes). If the issue is resolved before the delay expires, no ticket is generated.
Do Not Generate Ticket: No ticket is created, but the alert and any associated automations are still executed.

Note: The Generate Ticket with Delay option is available for conditions with a closing condition. For other conditions, only the options Generate Ticket and Do not Generate Ticket are visible. In case of alert threshold conditions, the Monitor Output value applies to all the thresholds configured.
