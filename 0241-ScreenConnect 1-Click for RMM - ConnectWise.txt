Overview

ScreenConnect 1-click enables you to connect to the remote servers and desktops without re-entering login credentials through a single click of the ScreenConnect icon.

1-Click for ConnectWise RMM

Sites can be opted-in for 1-click connection via Endpoints > Polices > select your Remote Access policy.

Note: You can review the ScreenConnect Integration with ConnectWise RMM article for information on creating this policy if needed.

Under the Remote Access Settings section, enable ScreenConnect 1-Click for Server, for Desktop, or for both.

Click Save.

Once a site is opted-in, 1-click connection will use server or desktop-specific login credentials stored in the site-level Passwords tab. If these credentials are unavailable, Automatic Server or Desktop Credentials will be used if enabled for the site. If neither of these server or desktop credential types are available, 1-click connection will not function, and the server-specific credentials will need to be manually entered to access ScreenConnect.

If multiple sets of credentials with user access are located in the site's Passwords tab, ScreenConnect 1-click will not function and the technician will be required to enter credentials.

Note: When credentials are configured, a user must have appropriate permissions set up through Access Control. A command gets executed in the backend if 1-Click option is enabled during the process of login.
