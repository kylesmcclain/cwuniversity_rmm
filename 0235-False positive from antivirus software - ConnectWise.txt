Introduction

In some case, users may see the ScreenConnect™ software falsely-flagged as malicious. This page explains how to handle false positives from antivirus software.

Background information

To prevent false detection, the ScreenConnect files are signed with code-signing certificates wherever possible. Some files cannot be signed since they are generated on-the-fly for each session. Occasionally, an antivirus may flag these files as suspicious.

White-listing ScreenConnect

In case your antivirus blocks ScreenConnect, you can try adding exceptions for the following files and directories.

Tip: See your antivirus's documentation for instructions for white-listing or creating exceptions for certain files.

Access agent files and directories

If an access agent is being flagged as suspicious, try white-listing the following directories:

Windows
C:\Program Files (x86)\ScreenConnect Client (xxxxxxxxxxxxxx)
Mac and Linux
/Applications/connectwisecontrol-xxxxxxxxxxxxxx

Or:

/opt/screenconnect-xxxxxxxxxxxxxx

Or:

/opt/connectwisecontrol-xxxxxxxxxxxxxx
On-premises server files

Note: For on-premises installations, these items are found in the \Bin folder of your ScreenConnect installation directory.

ScreenConnect.ClientSetup.exe
ScreenConnect.Core.dll
ScreenConnect.ClientBootstrap.exe
ScreenConnect.Client.zip
Reporting a false-positive

We recommend reporting a false-positive directly to your anti-virus vendor.
