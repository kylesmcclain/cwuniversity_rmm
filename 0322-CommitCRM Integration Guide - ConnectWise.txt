Overview

CommitCRM provides a complete view of customers, each stored as an account. Three main business functions—service and dispatching, sales and marketing, and billing—are supported by integrated modules.

The ConnectWise/CommitCRM integration automatically creates tickets in CommitCRM from alerts and service tickets generated by ConnectWise or from tickets created manually by the ConnectWise Network Operations Center (NOC). Integration includes the ability to link/create accounts to or from sites in ConnectWise and the ability to link/create assets to or from devices in ConnectWise. With two-way ticket integration:

ConnectWise’s tickets create tickets in CommitCRM.
Closing a ticket in CommitCRM closes the ticket in ConnectWise.
CommitCRM tickets can be assigned to ConnectWise’s NOC / Help Desk Service.
New tickets in ConnectWise can be created from CommitCRM tickets.
Comments for tickets can be sent from CommitCRM to ConnectWise.

Perform the following steps to integrate CommitCRM as your PSA system.

Note: 

Enabling a PSA integration disables email notifications through Notification Rules.
CommitCRM integration is not available in EU and AU.
1. Configure ConnectWise Settings

To allow CommitCRM to connect and sync with ConnectWise, a ConnectWise Partner Security Code is required.

To retrieve a code:

Log in to the ConnectWise Asio™ platform (ITSupport Portal) (control.itsupport247.net) with admin credentials.
Go to Settings > Account Administration > Integration > PSA Integration.
Select CommitCRM from the drop-down.
Copy the Security Code on the screen and select a resource to send email alerts to, then click Enable. The code locks to your account and is used to complete the configuration in CommitCRM.
Click Activate. You will receive the message CommitCRM Activated Successfully.

Note: It can take up to 24 hours for all settings to apply and all data to fully download. Plan to wait accordingly.

2. Set Up CommitCRM

After you have enabled the integration with ConnectWise, perform the following steps to set up CommitCRM.

Note: To avoid duplicative accounts, full integration with CommitCRM will be disabled by default until explicitly activated.

Back up CommitCRM database. Before starting to work, be sure to back up your CommitCRM database, including linking and importing sites and devices.

Initialize ConnectWise Security Code. After you configure ConnectWise settings and retrieve the security code, open Installation_DIR>\CommitCRM\RMM\CMTZenithLink.ini, paste the code as the value, then click Save.

Install CommitCRM RMM server service. The CommitCRM RMM server runs as a Windows service on your server. Be sure to complete the setup steps prior to installing the Windows service and allow the CommitCRM RMM server service in your settings.

To automatically install, log in to your server with admin credentials and run the installer from:

Installation_DIR>\CommitCRM\Utils\Install\InstallServerSideWindowsServices.exe

To manually install:

Right-click the command prompt icon and choose Run as Administrator.
Type the command: <Installation_DIR>\CommitCRM\RMM\CommitRMMServer.exe-install.

Note: The Installation_Dir must refer to a LOCAL server path and not a shared network name/path.

Wait for a success message to appear, indicating the service was installed successfully.
Go to Control Panel → Administration Tools → Services.
Verify the CommitCRM RMM server is displayed in the services list. (If not, make sure you are running the command prompt Window using Run as Administrator and try again.)
Right-click the start menu to open the services management window and select Startup type → Automatic Delayed Start for each time the server is restarted.
Download Site and Device Files. For ConnectWise tickets to be created for the correct account and asset, all ConnectWise sites should be linked to CommitCRM accounts, and all devices must be linked to assets. ConnectWise site and device lists should download automatically by the CommitCRM RMM server once activated; otherwise, an error message appears. Refer to CommitCRM Troubleshooting for details if the download fails.

Note: The ConnectWise integration must not be enabled at the CommitCRM side yet; only the service should be activated to download site and device details.

Link Accounts and Sites. If a site is not associated to any CommitCRM account, a temporary account is created to link a ticket. To avoid redundant creation of temporary accounts, all ConnectWise sites should be linked to equivalent CommitCRM accounts. Once all accounts are associated to sites, the system administrator can enable the integration. Section Five below contains additional details for linking accounts and sites.

Enable ConnectWise Integration. Enabling integration on the CommitCRM server will immediately start importing tickets from ConnectWise into CommitCRM:

Open <Installation_DIR>\CommitCRM\RMM\CmtZenithLink.ini.
Change the value for ProcessConnectWiseUpdatesQueue from N to Y.
Restart the CommitCRM RMM Server Service.
Uninstall CommitCRM RMM Server Service. If you ever need to uninstall the RMM server service:
Open the command prompt as an administrator.
Navigate to the \CommitCRM\RMM folder.
Type CommitRMMServer.exe-uninstall.
Wait for verification that the service was uninstalled successfully.
3. Enable Two-Way Ticket Integration

Once the integration is enabled, all tickets are managed fully in CommitCRM:

Tickets from ConnectWise automatically create tickets in CommitCRM.
Tickets can be created in ConnectWise from CommitCRM.
Tickets can be assigned to ConnectWise's NOC and Help Desk Service from CommitCRM.
Notes can be added to tickets in CommitCRM and sent to ConnectWise.
Tickets close from CommitCRM and automatically close in ConnectWise.

Note: Updates to ticket details will not update the ticket in ConnectWise, and closing a ticket in ConnectWise will not change the status in CommitCRM.

When a new ticket is created in ConnectWise (by a user or from an alert), the CommitCRM RMM server creates the ticket in CommitCRM. Ticket details include:

Description (taken from the ConnectWise ticket description).
Assignment (ticket is assigned to a defined account manager or default manager).
Notes (automatically added to the ticket with the creation date and time).
Asset (ticket is linked to the relevant asset. If the system cannot determine the asset, ConnectWise device details are added as a note to the ticket).

After the ticket is created, it is automatically added to the CommitCRM Tickets Inbox.

To create a ticket in ConnectWise (e.g. assign work to the ConnectWise NOC team), the ticket should be created in CommitCRM then transferred to ConnectWise:

Create the ticket in CommitCRM and open the ticket window.
Go to Tickets → Tools → Create/Update Ticket in ConnectWise.
Choose the assignee. A ticket can be assigned to:
NOC Team
Help Desk Service
CommitCRM
Add a Note if necessary under Comments.
Click OK.

Note: This process also can be used to reassign a ticket or add a note in ConnectWise. Updating ticket details (such as description or assigned technician) will not affect the ticket in ConnectWise.

4. Manage Tickets

When a ticket is assigned to the NOC or Help Desk Service, team members can return the ticket to CommitCRM marked Approval Required or Info Needed. The ticket goes into the Inbox, and the update is logged in the ticket's history as an audit entry. The note can be viewed in CommitCRM in the ticket history area.

Whenever a ticket is completed in CommitCRM, it automatically closes in ConnectWise. To close a ticket:

Open the ticket in CommitCRM.
Change the ticket status to completed (drag to a status in the toolbar or change the status manually).
5. Update ConnectWise Links

To open tickets from ConnectWise under the correct account and asset, create an association between sites and devices in ConnectWise to accounts and assets in CommitCRM by using the ConnectWise Links Manager Window.

Users must have integration privileges to manage ConnectWise links with sites and devices. To give a user the ability to update links, find the user's Privileges Group from the Employee Tab:

Go to File > Users and Privileges > Privileges.
Choose the relevant group from the drop-down menu and make sure the ConnectWise integration privilege is selected.

Note: CommitCRM RMM server service must be installed for ConnectWise sites and devices to be downloaded. Multiple sites in ConnectWise can be linked to the same account in CommitCRM since the sites may represent several physical sites for the same client.

Link Account to Site

In the Accounts Window, go to Accounts → Tools → ConnectWise Links Manager.

A Wizard opens, displaying an updated list of available un-linked sites from ConnectWise.

Select the site to link and click Next.
Find the parallel CommitCRM account to be linked to the external record by searching for the account by keyword or by using the account list recommended by the system.
Choose the account and click Link to Selected Account.

You can also opt to create a new account based on selected site details. When creating a new account, only the site name is copied to CommitCRM.

To cancel this link between the account and site, select the Linked External Records tab, find the account, and click Remove Link.

Link Asset to Device

In the Assets Window, go to Assets → Tools → ConnectWise Links Manager.

A Wizard opens, displaying an updated list of available un-linked devices from ConnectWise.

Select the device to link and click Next.
Find the parallel CommitCRM asset by searching by name or by viewing assets recommended by the system based on device name.
Choose the asset and click Link to Asset.

You can also opt to create a new asset based on selected device details. When creating a new asset, the device name, serial number, and manufacturer are copied. Other technical device details are copied to the Asset Note field.

To cancel this link between the asset and device, select the Linked External Records tab, find the asset, and click Remove Link.

Note: If a site or device is not linked to the equivalent account and/or asset in CommitCRM, the ticket will still be created with auto-generated names and details.

After the ConnectWise/CommitCRM integration is configured, the ConnectWise ticketing system (including email alerts) is disabled and ticket management is performed solely in CommitCRM. Tickets can still be viewed in ConnectWise as read-only. To receive email alerts for new ConnectWise tickets, consider using the Commit Alerts Server module, which sends email alerts for new updates in the CommitCRM system.

For troubleshooting or more details on the ConnectWise/CommitCRM integration, visit CommitCRM Setup.
