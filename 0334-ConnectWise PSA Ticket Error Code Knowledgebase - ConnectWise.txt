Error code : 1000

Details: The API Account or Member used for the integration is inactive, deleted, or doesn’t have permissions to perform this activity.

	

This error indicates that the API member used for the integration is no longer active or is deleted or doesn't have the necessary permissions to perform the transaction.

Impact:

The API member is used for executing all transactions in the ConnectWise PSA:

Create, fetch, or update ticket information when they are updated in the ConnectWise RMM
Create or update Configuration items in ConnectWise PSA
Fetch contact information when they are created, updated, or deleted in ConnectWise PSA
Create or update the agreements for the additions count for endpoints

Troubleshooting steps:

Please validate the API member used for the integration is active. Review the ConnectWise PSA Integration documentation.
In case the API member is inactive or deleted, please refer to the Details Tab - API Members documentation for help creating a new member.
Make sure that there are no restrictions applied to the member account for updating tickets on specific boards. If the ticket is moved to a board to which the API member doesn't have access then the specific tickets will fail to sync. 

After the necessary permissions are provided, Please try resyncing the tickets from the PSA Integration section to force the tickets to resync.

If the issue persists, please contact the PSA Support team for assistance via an online chat session. Follow the steps below to initiate a chat session,

Login to the ConnectWise Asio™ platform.
Click Support.
Select the option - PSA Support.
Select ConnectWise PSA.

Error code : 1001

Details: The RMM Integration doesn't have the appropriate permission to complete the transaction.

	

This error indicates that the integration doesn't have appropriate permission to complete the transaction. This error can occur while performing multiple different operations like:

Creating a ticket
Updating a ticket
Creating a configuration item
Updating a configuration item
Updating an agreement

Troubleshooting steps:

Tickets:

Make sure that the company status allows tickets to be created or updated 
Make sure that the API account is not restricted to performing transactions on a specific company or service board

Configuration item:

Make sure that the company status allows configuration items to be created or updated

Agreements:

Make sure that the company status allows agreement changes to be performed

After the necessary permissions are provided, please try resyncing the tickets from the PSA Integration section to force the tickets to resync.

If the issue persists, please contact the PSA Support team for assistance via an online chat session. Follow the steps below to initiate a chat session,

Login to the Asio platform.
Click Support.
Select the option - PSA Support.
Select ConnectWise PSA.

Error code: 1012

Ticket Sync Policy is disabled for Site

	

This error indicated that ConnectWise PSA Ticket Sync Policy is not mapped with Company/Site.

Impact:

Tickets from RMM/PSA will get not synced as the ticket sync policy is disabled for the site

Troubleshooting:

Make sure that there is a valid ConnectWise PSA Policy assigned to the Site.
Please refer to the ConnectWise PSA Policies Documentation for more information on creating and applying the Policy
