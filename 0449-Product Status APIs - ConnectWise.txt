Overview

These APIs allow product vendors to push monitoring data from their products per endpoint into the ConnectWise RMM. This data is shown against each device in a dedicated device list of the Asio Portal. 

Get Status Schema
Post Endpoint Status

The following steps must be performed in order to push data: 

The vendor will have to work offline with ConnectWise to create a data schema for their product. An example schema is shown with the GET Statuses Schema API.
Data for the product will then be accepted only as per the defined schema.
Data schema can be edited on request. 

Method for uniquely identifying an endpoint to push data for: 

The Get Clients, Sites, and Endpoints (with Endpoint Details) APIs will give Endpoint ID details. Vendors can post data using the Endpoint ID received via these APIs. There are two methods via which vendors can identify the Endpoint for which they need to push data: 

Method 1: (Recommended Method) Endpoint ID is exposed by the ITS Platform agent locally in each partner's machine. Vendors can directly fetch this Endpoint ID from the machine via their products and systems. Vendors can fetch the Endpoint IDs from the following sources in the machine: (Recommended Method) 
For Windows machines, Endpoint ID is available in the Registry "HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\ITSPlatform".
For LINUX machines, Endpoint ID is available in the config file "/opt/itsplatform.conf".
For MAC machines, Endpoint ID is available in the plist file "/Library/Preferences/com.itsplatform.plist".
Method 2: Using the Get Endpoint Details API, certain hardware parameters for the machines are available (like System Name, Serial number, MAC address, etc.). Vendors may use a combination of one or more of these parameters to uniquely identify a machine.

Post Endpoint Status
Request

Sandbox: POST https://openapi.sandbox.use1.cwcmddev.cwnet.io/v1/clients/clientID/sites/siteID/endpoints/endpointID/statuses 

Production: POST https://openapi.service.itsupport247.net/v1/clients/clientID/sites/siteID/endpoints/endpointID/statuses 

Parameters
clientID	Example clientID: 2lUrtru6uvuVs7 	String
siteID	Example siteID: wOCDc3uZu7upuZ 	String
endpointID	Example endpointID: 4a9caa07-d45a-4c14-aa36-8c1c26e7a029 	String
Headers

Authorization:Bearer i9OpjK8IBYYNbV9VGJo6APG224I 

Response

201

Headers

Content-Type:application/json 

Body

}
