Overview

May 23, 2018 

With the May 2018 release, we have provided an additional option to send the PSA Ticket ID in the ConnectWise System. This enhancement allows the NOC/HD team to reference ticket ID numbers when partners or end-users call. Queries can be run using either the Asio Ticket ID or PSA Ticket ID to provide additional assistance. This enhancement facilitates better communication with the NOC Team and also enables you to create automated emails with PSA Ticket IDs for your Help Desk end clients that can be used for reference when they contact the Help Desk team for assistance.

HelpDesk Tickets once generated by the HD team now have Contact details available that can be used for mapping users to tickets in the PSA system. The Contact Id field is available with regular ticket requests.

Additionally, we have added a GetContact operation that enables you to pull all the Contact Information stored and available in the ConnectWise System for the site (the contact details should be updated in the Client profile section):

Id (Unique for each contact)
First Name (First Name of contact)
Last Name (Last Name of contact)
Job Title
Email Address
Primary Phone Number
Primary Phone Number’s Extension
Secondary Phone Number
Secondary Phone Number’s Extension
Mobile Phone Number
Unique ID for Site

Below are the operations supported in ConnectWise's public ticketing API.

GetTicketStatuses

Gets a list of all ticket statuses from the ConnectWise system.

Status

	

Status Details




Acknowledged

	

This status would be used by ConnectWise when we would acknowledge the ticket being received on our end. Once the ticket has been acknowledged it would be assigned to a tech to work on the same.




Approval Needed

	

This status would be used by ConnectWise when we would need some approval from Partner to continue working on the issue further.




Approved

	

This is a partner status and it would be used by a partner when they are providing some update on a ticket or reverting with approval to work on the issue.




Closed

	

This status would be used by partner and NOC both. Ticket once closed in ConnectWise cannot be reopened.




Info needed / Work Assigned

	

This status would be used by NOC when we would need additional information from your team to continue working on a issue.




Interim Notification

	

This status would be used by ConnectWise when we are continuing to work on a issue but wanted to provide you with a interim status of the work performed and next action plan.




New

	

This status would be used by us and partner. This notifies a new ticket either on either systems.




NOC Completed

	

This status would be used by ConnectWise when we would be updating the ticket once the work requested on our end has been completed. This doesnâ™t correspond to Complete as partner can still update on the ticket they feel some work was missed or you would like request for additional changes for issue.




Not Approved/ Further Info Needed

	

This will be used by partner when he does NOT approve for work to be done by ConnectWise team or he needs more information from us about the issue.




Pending

	

This status is used by the partner when he is awaiting some information on their end before we can commence the work further.




Schedule

	

This status would be used by NOC when we would be scheduling a ticket to be worked at a specific interval or timeframe.




Task Missed/Not Completed

	

This status will be selected when the ConnectWise team was unable to complete a ticket.




Waiting Customer

	

This status would be used by the partner when they are either waiting on some third party or end client to get confirmation for NOC to work on the issue further.




Work In Progress

	

This status would be used by NOC when we would start working on a specific issue.

Note that the following Status id numbers are internal for partners and are not used by ConnectWise System:

Status id: 205
Status id: 206
Status id: 207
