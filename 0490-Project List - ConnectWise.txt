Prerequisites

For Server Projects, in addition to any prerequisites listed below, please make sure you have the following:

Administrator (Root) or Domain Administrator credentials.
The RMM Agent installed on the server in question.
Software licenses for new installs or migration.

Active Directory Migration

When you want to upgrade your Active Directory to a newer version, our Tech Advantage team performs the following steps:

Health check of existing domain environment
Forest Schema preparation
Domain Schema preparation
Install Active Directory on Windows Server 2008/R2/2012
Transfer FSMO roles to new ADC
Transfer DNS, DHCP, WINS roles
Keep old Domain Controller offline for a week and observe the environment for any impact
Decommission the old Domain Controller
In addition, our team can customize the migration based on your client’s requirements.
Active Directory Setup

For Active Directory Setup, our Tech Advantage team performs the following steps:

Health check of existing environment
Install Active Directory and DNS server role
Verify replication
Configure the PDC emulator for the forest root Authoritative Time server
Configure DC as Global catalog server
Install DHCP, WINS roles (optional)

In addition, our team can customize the setup based on your client’s requirements.

BlackBerry Enterprise Server for Microsoft Exchange Migration

For the Blackberry Enterprise Server for Microsoft Exchange Migration, our Tech Advantage team performs the following steps:

Review and scope production environment
Detect issues and recommend actions
Assign permissions for the new BlackBerry service account according to Windows (AD)
Install the Blackberry Application
Create new database with existing users configured
Perform migration from old Bes to new Bes server
Verify functionality
Decommission the old server

In addition, our team can customize this project based on your client’s requirements.

BlackBerry Enterprise Server for Microsoft Exchange Setup

For the Blackberry Enterprise Server for Microsoft Exchange Setup, our Tech Advantage team performs the following steps:

Health check of existing environment
Creating BlackBerry Service account (besadmin) with permissions according to the Windows (AD)
Assign permissions to BlackBerry service account according to the Exchange server version
Install Blackberry Application
Create BlackBerry configuration database (besmgmt) or using the existing database
Configure and check the permissions for the BlackBerry configuration database
Verify the necessary ports for BES are open on the server
Activate the users from server-side
Verify the windows account (besadmin) can access the users mailbox in Exchange server

In addition, our team can customize this project based on your client’s requirements.

Custom Server Project

For unique server projects not listed, select this option and complete the required fields instead of submitting the request via email to the NOC. This step will provide the information the NOC needs upfront to begin working on the project as efficiently as possible.

Exchange Server Migration

For the Exchange Server Migration, our Tech Advantage team will migrate and test the Exchange environment and complete the following:

Review of current production environment
Pre-deployment analysis to ensure Exchange compatibility
Prepare environment for an exchange migration
Install and configure new version Exchange into client environment
Run a complete test suite on the Exchange server
Migrate existing Mailboxes and public folders
Configuration of Active Sync and OWA
Autodiscover Configuration*
Testing of complete new environment*
Decommission the old server

Note: Since the auto discovery feature is being migrated to a new server, the SAN certificate needs to be modified in order to generate a new certificate; ConnectWise partners need to provide details so that NOC can generate a certificate file, which the partner would use to generate a certificate from their provider such as GoDaddy or VeriSign. If you select a third-party SAN Certificate, please provide details below for generating a certificate request file.

Organization
Organization Unit
Country/Region City/Locality State/Province

In addition, our team can customize this project based on your client’s requirements.

Note: For Exchange 2003 to 2013 migrations, if there is no direct upgrade path, additional charges may be applied due to an increase in service hours.

Physical to Virtual (P2V) Machine Conversion using VMware vCenter Converter

For the Physical to Virtual (P2V) Machine Conversion using VMware vCenter Converter, our Tech Advantage team performs the following steps:

Assessment of existing physical server for storage, network, and compute capacity
Stage vCentre for new virtual machine requirement for network (port group), compute (vCPU) and storage (vmdk) based on requirement
Use P2V converter tool to perform block level replication to destination virtual machine instance
Health check for network connectivity; verify required services and functionality for application
Shut down old server
Bring destination server in production and check errors and warning; perform required corrections; check server for optimal performance

In addition, our team can customize this project based on your client’s requirements.

Server Service Pack Installation

For the Server Service Pack Install, our Tech Advantage team performs the following steps:

Download Service Pack
Perform backup of Registry, Applications etc.
Install the Service Pack
Reboot the Server
Check for any error messages in Event Logs
Ensure all applications are working properly; e.g., if we installed Exchange Service Pack, we will ensure all Exchange services are up and running and test mail flow and check mail queue to ensure everything is running properly.

In addition, our team can customize this project based on your client’s requirements.

Small Business Server Migration

For the Small Business Server Migration, our Tech Advantage team performs the following steps:

Perform check of existing small business server, check upgrade prerequisites
Upgrade Active Directory to support new SBS version
Stage the small business server installation in migration mode
Configure new server with features required; move existing configuration from old SBS Server (DHCP Server, Offline Address Book, recipient email address policy)
Move mailboxes and replicate public folder to new server
Ensure routing of emails is changed to new server
Ensure new version of small business server is functional with all roles required such as Active Directory, DHCP, active Sync, Outlook anywhere etc.

In addition, our team can customize this project based on your client’s requirements.

Small Business Server Setup

For the Small Business Server Setup, our Tech Advantage team performs the following steps:

Configure Active Directory, DHCP Server service with group policies setting if required
Configure Small business Server with features and policies required for Exchange role
Run Windows update to make sure security patches and upgrades are proper
Test functionality of features requested such as Outlook web access, Outlook anywhere, Active Sync, remote web workplaces as requested
Test mail flow from multiple domains
Run best practice analyzer for small business server to perform health check and do correction if required
Verify end-user connectivity to Exchange Server is functionally working

In addition, our team can customize this project based on your client’s requirements.

VMware vCenter Server Setup

For vCenter setup, our Tech Advantage team performs the following steps:

Verify prerequisites to deploy vCenter
Install vCenter with required prerequisites such as .Net, SQL database configuration
Add all ESX hosts to vCenter inventory and verify all information related to ESX hardware, events and performance of ESX host server are visible
Configure and tests vCenter features that were part of installation request for high availability
Check VMware network setting as per best practice for network redundancy, management network and storage connectivity
Check for errors and warning under alarms and correct those if required
Make sure update is done on vCentre suite for any bug fixes if applicable

In addition, our team can customize this project based on your client’s requirements.

Webroot Deployment
All communications with the MSP are updated in the ticket.
The ticket is scheduled as agreed upon with the MSP.
When work is complete, the ticket is marked as NOC Completed.

For Webroot deployment, our Tech Advantage team performs the following steps:

Configures the Webroot console (Creating sites, policies, exclusions etc.)
Installs Webroot using scripts and a defined schedule.
Identifies Windows systems and MAC systems using ITSupport Portal.
For Windows desktops, scripts used are run at next agent check-in so that all the agents/desktops that are not reporting (most likely offline) will get Webroot the next time they are online. This job runs for 7 days for it to capture as many online desktops as possible.
For MAC systems, the NOC manually installs Webroot.
Webroot install using site selected policy.
NOC observes for 45 minutes to check if there are any valid/invalid files being reported as threats by Webroot.
If there are threats reported, the NOC contacts you for information about the files being detected as threats.
If you provide information then the NOC validates/invalidates the files. 
Once finished, the NOC uninstalls the existing AV solution using a script. 
If threats were not reported, the NOC proceeds with uninstalling the existing AV solution using a script.
If applicable, the NOC uninstalls the Viper console from servers.
Windows Operating System Installation on Hypervisor

For the Windows OS on VMware or Hyper-V - Installation and Configuration, our Tech Advantage team performs the following steps:

Configure virtual desktop or Server instance with required settings based on requirement i.e. vCPU, vdisk, memory settings
Install operating System, hyper-v client, service packs and run Windows update for security updates and optional updates
Perform Windows product activation by adding serial number
Configure Windows for security and desktops settings as per requirement mentioned
Install any roles on Server as per requirement mentioned

In addition, our team can customize this project based on your client’s requirements.

Windows Server Hyper-V Setup

For the Windows Server Hyper-V Infrastructure - Installation and Configuration, our Tech Advantage team performs the following steps;

Verify network infrastructure before implementing Hyper-V setup, e.g. network, storage, Hyper-v host server for virtualization support
Add Hyper-V roles and configure Virtual Networks with selected LAN adapters that will be shared with guest sessions, management and live migration (if clustered)
Run Windows Update to confirm that all patches and updates are installed and applied for Hyper-V server
Configure Hyper-V server for storage configuration and cluster configuration
Run best practice analyzer for Hyper-V to check for optional configuration

In addition, our team can customize this project based on your client’s requirements.

Windows Server Hyper-V with SCVMM Setup

For the Windows Server Hyper-V with SCVMM, our Tech Advantage team performs the following steps:

Install and configure System Center Virtual machine manager with prerequisites such as winrm, .net framework, PowerShell, IIS, Microsoft baseline configuration analyzer and SQL server settings
Install VMM administrative console and add windows Hyper-V host
Run system center configuration analyzer and verify that the settings are optimal

In addition, our team can customize this project based on your client’s requirements.
