Overview

Below are prerequisites and other ticketing API basics to review before configuring your API operations.

Prerequisites
Must have a Super Admin account in the Asio™ platform.
Enable the PSA Integration option per the PSA system used. PSA Integration enablement options are available in the Asio platform by navigating to Settings > Account Administration > Integration > PSA Integration.
Authentication

To enable the Asio platform to properly and safely authenticate API calls, make sure that the Integrator User and Callback details are up-to-date in the PSA Integration.

Callback Mechanism

A callback is a programming practice of transmitting data to another program. An Asio ticketing API callback occurs when any create or update action is dispatched on a ticket object in Asio.

Use the Callback URL field in the PSA Integration page to specify a standard or custom callback URL. When the callback in Asio is triggered, the Ticket ID and the Action will be added to the URL by GET/Post method, as follows:

action = <new/update>
NOCTicketID = <numeric value of NOC Ticket ID>
NOCTicketNotesID = <numeric value of NOC Ticket Notes ID, if added in Ticket Object>

Ticket Source

ConnectWise tickets are generated in the following ways:

Automatically by RMM agent
By ConnectWise NOC or Help Desk Services technicians
By MSPs via public ticketing API
By MSPs directly from ITSupport Portal
Ticket Types

ConnectWise tickets can be:

Resource-specific
Site-specific
Adhoc, created by the MSP and not necessarily affiliated with any ConnectWise site or resource
