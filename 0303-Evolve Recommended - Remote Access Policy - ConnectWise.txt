Overview

This article covers the recommended settings to employ for your Remote Access Policy. These best practice settings for different RMM configurations are based on the feedback and analysis of policies used by partners.

Policy Details
Policy Usage	Remote access policies allow the partner to define whether the ScreenConnect® application should be deployed on the endpoint. The partner can define whether ScreenConnect can be deployed on server or desktop machines.
Policy Type	Remote Access
Policy Name	Evolve Recommended - Remote Access Policy
Policy Applies To	

These settings will be applied to the following types of agents:

RMM Enabled Windows Server

RMM Enabled Linux Server

RMM Enabled Desktops

RMM Enabled MAC
