Error code : 1000
 

Details: The API Account or Member used for the integration is inactive, deleted, or doesn’t have permissions to perform this activity.

	

This error indicates that the API member used for the integration is no longer active or is deleted or doesn't have necessary permissions to perform the transaction.

Impact:

The API member is used for executing all transactions in the ConnectWise PSA :

Create, fetch, or update ticket information once they are updated in the ConnectWise RMM
Create or update Configuration items in PSA
Fetch contact information when they are created, updated, or deleted in ConnectWise PSA
Create or update the agreements for the additions count for endpoints

Troubleshooting steps:

Please validate the API member used for the integration is active. Review the ConnectWise PSA Integration documentation.
n case the API member is inactive or deleted, please refer to the Details Tab - API Members documentation for help creating a new member.
Make sure that there are no restrictions applied to the member account for updating tickets on a specific boards. If the ticket is moved to a board to which the API member doesn't have access then the specific tickets will be failed to sync. 

After the necessary permissions are provided, Please try resyncing the tickets from the PSA Integration section to force the tickets to resync.

If the issue persists, please contact the PSA Support team for assistance via an online chat session. Follow the steps below to initiate a chat session,

Login to the ConnectWise Asio™ platform.
Click Support.
Select the option - PSA Support.
Select ConnectWise PSA.

Error code : 1001

Details: The RMM Integration doesn't have the appropriate permission to complete the transaction.

	

This error indicates that the integration doesn't have appropriate permission to complete the transaction. This error can be surfaced while performing multiple different operations like:

Creating ticket
Updating ticket
Creating configuration item
Updating configuration item
Updating agreement

Troubleshooting steps:

Tickets:

Make sure that the company status allows tickets to be created or updated 
Make sure that the API account is not restricted to performing transactions on a specific company or service board

Configuration item:

Make sure that the company status allows configuration items to be created or updated

Agreements:

Make sure that the company status allows agreement changes to be performed

After the necessary permissions are provided, please try resyncing the tickets from the PSA Integration section to force the tickets to resync.

If the issue persists, please contact the PSA Support team for assistance via an online chat session. Follow the steps below to initiate a chat session,

Login to the Asio platform.
Click Support.
Select the option - PSA Support.
Select ConnectWise PSA.

Error code : 1012

This error will be returned when the ConnectWise RMM is attempting to either create or update the configuration item but there is a required field mapped which is not available with the sync.

	

This error will be returned when the ConnectWise RMM is attempting to either create or update the configuration item but there is a required field mapped which is not available with the sync.

Troubleshooting Steps:

Error indicates that there is a mandatory field mapped for the creation/update of a configuration item
These fields are not pushed via the PSA but due to configuration in PSA, system will reject the creation as mandatory field is not available
Remove such field from mandatory field or set some default value so that sync can complete the transaction

After the necessary changes are done in ConnectWise PSA for the configuration item, please try resyncing the tickets from the PSA Integration section to force the tickets to resync.
