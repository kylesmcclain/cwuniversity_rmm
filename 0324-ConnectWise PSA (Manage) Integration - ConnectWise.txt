Overview

ConnectWise RMM™ features the ability to integrate with the ConnectWise PSA™ system to sync tickets, assets, configuration items, billing data, and contacts.

 To access this feature, navigate to the ConnectWise Asio™ platform > Settings > Account Administration > Integration > PSA Integration.

User Access

Refer to the following table for a complete list of user access:

User Role	Super Admin	Manager	Technician	Office Staff	Sales	Client Site Manager	Client End User
PSA Integration	Always ON	ON by default	ON by default	Always OFF	OFF by default	Always OFF	N/A

 

Please refer to Changing Access Rights for information on changing the user access and permissions.

Ticket Purging Policy

Our PSA integration system follows a purging policy for the removal of failed tickets from the sync program. We are following this policy to improve the performance of the overall system.

We are purging failed tickets from the PSA integration sync program based on the following conditions:

Both incoming and outgoing tickets with transactions or conversations that have been ongoing for one year (365 days) from the creation date will be removed. If the PSA integration system receives an update on a purged ticket, it will produce a new ticket based on the most recent update, but the previous notes will be purged and will no longer be available.
Incoming and outgoing tickets that fail due to incorrect authorization or incorrect keys will be removed from the fault table after 45 days of the ticket creation. 
Incoming and outgoing tickets which were updated by the NOC team as closed after 30 days of inactivity will be removed from the PSA integration sync program after a 15 day buffer period. In other words, tickets closed by NOC will be removed after 45 days from creation.
