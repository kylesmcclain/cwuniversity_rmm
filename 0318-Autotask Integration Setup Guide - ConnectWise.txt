Note:

The Password/secret is only enabled or active when you save the API user. Using a custom password or auto-generated key and secret is only effective when the user is saved.
You can copy and maintain the username and password for future reference.
Please make sure that the Integration vendor is correctly selected to Continuum - Autotask API RMM Integration. Selecting an incorrect vendor would impact the integration and might not work correctly.
b. Create Product Categories

Product categories must be manually created so that the Configuration items can be mapped correctly to them. The Asio platform needs the following categories to be created in the Autotask:

Server
Desktop
Virtual Host
Backup

Review the Autotask documentation for Adding Product categories. 

c. Create Configuration item types

ConnectWise RMM will create configuration items in the AutoTask PSA system to map the endpoints managed by the RMM system. These configuration item types are for all the different devices managed in the system like Server, Desktop, Mobile and backup devices.

Windows Server
Linux Server
Macintosh Server
Windows Desktop
Macintosh Desktop
VMware Host
ConnectWise BDR

Review the Autotask documentation reference for creating Configuration item types.

Note: Currently, we are fetching the install date for assets in UTC (Coordinated Universal Time) format for Autotask integration.

d. Create Ticket Queues

Tickets queues are repositories that hold tickets generated from the NOC and HelpDesk teams. Any tickets that will be created or updated by the NOC team will be automatically updated against these queues. The following queues should be created in Autotask:

NOC
Service Desk

Note:

Both the NOC and Service Desk queues should be set to active.
The Queue names must be entered exactly as shown above.
The NOC and Service Desk queues must be used for ticket assignments to the ConnectWise NOC or Service Desk only.  If you are using those queues for other purposes in Autotask, please create new queues and keep the NOC and Service Desk queues for ConnectWise ticket use only.
Tickets created by the NOC and the Help Desk are created with the Ticket Type of Incident in Autotask, unless it is specified as a Problem by the NOC or Help Desk technician.

Incoming tickets from Autotask are created in ConnectWise with the ticket type Service Request.

Review the Autotask documentation for creating Ticket Queues.

e. Create Ticket Sources

Tickets generated by the ConnectWise NOC and HelpDesk team will be mapped to the sources to identify the ticket origin or updated scenarios. The following sources should be created in Autotask.

NOC
Service Desk

 

Note:

Both the NOC and Service Desk source should be active.
The source names must be entered exactly as shown above.

Review the Autotask documentation for Creating Ticket sources.

Step 2: Enable the integration in the Asio platform 

Next you need to enable the PSA Integration in the ConnectWise Asio platform in order to use the API user created in Autotask. 

Navigate to the following location:

ConnectWise RMM: Asio platform > Settings > Account Administration > Integration > PSA Integration

Select AutoTask from the drop-down list. The system prompts you to enter the Autotask username and password:

The Autotask user name is the Username (key) for the API user

The Autotask Password is the Password (Secret) on the API user screen

Click Enable and continue with the setup process.

After the Autotask user name and password is accepted, it displays a unique URL (Callback URL) and other information.

Step 3: Create your Extension Callout

After the PSA is enabled, the Asio platform generates a unique call-back URL for your instance. Update this information in Autotask portal so that Asio is notified when there are changes made to tickets in the Autotask portal.

In the Autotask portal, navigate to .Admin > Extensions & Integrations > Other Extensions & Tools > Extension Callout (Tickets).
Click New Extension Callout to open the screen to enter information.
Update the following information on the screen to complete the extension callout creation:
Name: You can define any name of your choice for the extension
URL: Add the URL from the Asio platform that was generated as a part of Step 2. Make sure that there are no leading or trailing spaces while copying the URL.
Set the Active checkbox.
Username, password, and Ticket User-Defined field should be left blank
HTTP Transport Method should be set as GET
Save the extension callout

Review the Autotask documentation for Extension Callout (Tickets).

Step 4: Add a Workflow Rule to trigger the extension callout.

Once the Extension callout has been created, it needs a trigger defined to start sending the ticket information to the Asio platform. Use the following steps to define and create the workflow rule for the extension callouts:

In the Autotask portal, navigate to Admin > Automation > Workflow Rules > Workflow Rules.
Click the Service Desk Tab.
Click New.
In the Workflow Rule Editor Window, update the following fields:
Workflow rule name: This is the name for the workflow which will trigger changes in the Asio platform.
Description: This is optional field but you can use this field to add information or a description for the callout.
Make sure that the Active checkbox is enabled.
Under the Events section, make sure that the following options are selected:
Created by (Value to be set as Anyone)
Edited by (Value to be set as Anyone)
Note Added by (Value to be set as Anyone)
Time Added by (Value to be set as Anyone)
Under the Actions section, make sure that the following options are updated:
Execute Extension Callout: Select the new extension callout that was created in Step 3.
Save the Extension callout editor.

Review the Autotask documentation for the Workflow rule for the Extension Callout.

Step 5: Complete the setup in the Asio platform.

Navigate to the following location:

ConnectWise RMM: Asio platform > Settings > Account Administration > Integration > PSA Integration.

Complete the remaining mandatory fields:

Set Material Cost Code

Select a Default Product Material Cost Code based on the value that should be used for the integration. Refer to the Autotask documentation for details on the Material cost code.

Set Default Site for tickets

Select a Default Site for Tickets. These drop-down values represent the sites that are defined in the Asio platform. This are used for mapping tickets that are created as Adhoc (No site attached but the ticket is generated at Partner level).

Map Ticket Statuses

Click Map Ticket Statuses to select how ConnectWise RMM fields map to Autotask fields and vice versa. This is used for the ticket sync between ConnectWise RMM and Autotask.

ConnectWise RMM ticket Statuses to AutoTask Ticket status: This is used to map the ticket statuses in Autotask when a ticket in ConnectWise RMM is created or updated.
AutoTask ticket Statuses to ConnectWise RMM status: This is used to map the ticket statuses in ConnectWise RMM when a ticket in Autotask is created or updated.

 

Notes:

Each status should be mapped.
New statuses added in autotask and not mapped result in incorrect ticket mapping or sync.
We strongly recommend that you map ConnectWise RMM to Autotask and vice versa with same status to avoid ticket looping issues.

Note: If you are facing issues with ticket sync with errors like Ticket Status is not available or Ticket priority is not available, then resave your Ticket Status / Priorities from Asio and click the Test Configuration button provided in Asio. After that, trigger a resync from Asio to sync the failed tickets.

Map Priorities

Click on the Map Ticket Priorities button to select how ConnectWise RMM fields map to Autotask fields and vice versa.

Using the drop-down menus, map the ConnectWise RMM Ticket Priorities to the Autotask Ticket Priorities and vice versa. This allows you to use custom Priorities names in Autotask and map them with the ConnectWise RMM Ticket priorities.

Note: We strongly recommend that the Status mapping directions from Autotask to ConnectWise RMM and ConnectWise RMM to Autotask are mapped correctly to avoid looping.

For example: If the ConnectWise RMM priority of Emergency is mapped with the Autotask priority of High, then the same should be on reverse mapping (Autotask to ConnectWise RMM).

Note: If you are facing issues with Ticket Sync with errors like Ticket Status is not available or Ticket priority is not available then resave your Ticket Status / Priorities from Asio and click the Test Configuration button provided in Asio. After that, trigger a resync from Asio to sync the failed tickets.

Site Mapping

Click on Map sites to open the screen where you can map the ConnectWise RMM site with the Autotask Company Name.

Notes:

Only Autotask customer company types will be displayed in this list. Non-customer company types will not be displayed.
Currently, the system will allow one-to-one mapping of ConnectWise RMM sites with Autotask companies. 
You can skip some ConnectWise RMM sites to be mapped with Autotask companies, but doing so will not sync any tickets, contacts, assets and billing information against them.
Error Notification Emails

Select one or more users from the Asio platform who should be notified when there are issues observed with the PSA integration with Autotask.

Sync Configuration Items
Select device types and fields of configuration items to sync.
Select the checkbox to select a device type
Select device-specific fields in the table below. Leave checkboxes deselected for fields you do not want to sync (warranty info, for example).
Click Save Configuration to apply your settings.

Billing

Specify the billing configuration mapping against the ConnectWise RMM sites to be pushed in the corresponding Contracts and Service / Service Bundles.

From the Billing window, locate a Site or Company in the list and click on the corresponding + button. This will expand the billing configuration options for the site.
Use the drop-down menus to make the following selections for each device type (Servers, Desktops, and ConnectWise Backups) as desired:
Contract: The contract to apply to the device type
Service / Service Bundle: The service / service bundle to apply to the device type.

Note: Contract and service / service bundle combinations can only be used once. When a contract and service / service bundle combination is assigned to a device type, it cannot be assigned again to a different device type.

Device Count Increase or Decrease: This feature lets you define how changes in device count should affect the selected contract:
Do not change the number in the contract: The contract's device count will not be updated when devices are added or removed. We recommend using this option if you have a fixed devices cost to be applied for the client.
Change the number in the contract: The contract will be automatically updated to reflect the adjusted device count each time a device is added or removed.

Note:  If you select Change the number in the contract, changes in device count will be collected and applied when the next daily synchronization occurs.

The Servers device type also provides additional options for adding virtual machines to the server device count:
Include VMware Hosts in the server count
Include Hyper-V Hosts in the server count

Note: Billing data will be pushed in the corresponding contracts when there is any changes made on the configuration items (Addition or removal). Changes will be added in the corresponding Autotask as prorated changes.

Devices once removed with the ConnectWise RMM monitoring agent will be updated as inactive in Autotask and the counts will be removed.

How to Create an Exception

The Exceptions section pre-populates with a blank row you can use to create your first Exception. If an Exception already exists, click on the + Add Exception link directly below the table to create additional Exceptions.

To create an Exception, complete the following fields:

Select Product: Use the menu to select Server, Desktop, or ConnectWise Backup.
Select devices: Use the check boxes to select individual machines. Note: If this field is blank when you click on the menu button, click inside the empty field to generate the device list.

Select Contract: Select the contract that should be applied to these devices.
Select Service/Service Bundle: Select the service/service bundle to apply to these devices.
Maintain/Reduce contract unit: Select the contract action that should be applied if any of the selected devices are removed:
Maintain contract unit when removed: Maintain the device count in the contract when any of these devices are removed from the site.
Reduce contract unit when removed: Decrease the device count in the contract when any of these devices are removed from the site.

You can set exceptions in the billing records as required for each individual endpoints managed by the ConnectWise RMM.

ConnectWise RMM to Autotask Ticket Note Mapping 

Specify how notes added in the ConnectWise RMM tickets to be displayed in Autotask ticket note type. Notes should be classified for both the NOC/HelpDesk/MSP and whether notes should be accessible to Internal Users Only, All Autotask Users, or Both once the notes sync with the Autotask portal.

Autotask to RMM Notes Syncing

Sync Ticket notes that were published for: Select which ticket note types should be synced to the ConnectWise system: notes published for Internal Users Only, All Autotask Users, or Both.

When a ticket is assigned to the NOC or Help Desk Service

Only sync last note: Only the last note added to a ticket in Autotask is synced when the ticket is assigned to NOC/HD.
Sync all notes: Syncs all notes in the ticket. See below for how this functionality operates based on the ticket creation date.

Step 6: Activation and Test Configuration 

Click Activate button in the Asio platform after the settings are finalized.

Now, click Test Configuration to make sure the configuration is set up properly. 

The system validates all the prerequisites required for the integration to be completed.
