Error code: 1000

Details: API Account/Member used for the integration is inactive/deleted/doesn't have permissions to perform this activity

	

This error indicates that the API member used for the integration is no longer active or is deleted or doesn't have necessary permissions to perform the transaction.

Impact:

API member is used for executing all transactions in ConnectWise PSA:

Create/fetch/update tickets information once they are updated in ConnectWise RMM.
Create/update configuration items in Autotask PSA.
Fetch contact information when they are created/updated/deleted in Autotask PSA.
Create/update the agreements for the additions count for endpoints.

Troubleshooting steps:

Validate the API member used for the integration is active. Refer the documentation for API keys: ConnectWise PSA (Manage) Integration Setup Process - ConnectWise .
In case the API member is inactive/deleted, refer the following ConnectWise PSA documentation to create a new member: Details Tab - API Members.

After the necessary permissions are provided, try resyncing the billing data from the PSA Integration section to force resync.

In case the issue persists, contact the PSA Support team for assistance via online chat session. Follow these steps to initiate a chat session:

Log in to the ConnectWise Asio™ platform.
Click Support (Top-right)
Select option PSA Support
Select next option ConnectWise PSA
