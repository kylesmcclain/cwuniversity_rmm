Below are two examples of ticketing API operations with data included.

Example 1: CreateTicket

This example shows the request and response body of the CreateTicket operation. This operation accepts the “Ticket” object and returns the “TicketResponse” object.

The TicketResponse object has two properties:

<Results>: If operation succeeded, API will return the “Ticket” object here.
<Error>: If operation failed, API will return an error code and error message here. For the full list of error codes and descriptions, see Public Ticketing API Exceptions.
Request Body

<soapenv:Envelope xmlns:soapenv="http://www.w3.org/2003/05/soap-envelope" xmlns:v1="http://www.continuum.net/its/v1_0">

<soapenv:Header>

<UserID xmlns="TicketService">Testintegrator1</UserID>

<Password xmlns="TicketService">XXXX</Password>

<TokenID xmlns="TicketService">53H48H48H48H49H55H50H56</TokenID>

</soapenv:Header>

<soapenv:Body>

<v1:CreateTicket>

<v1:request>

<v1:SiteID>50016263</v1:SiteID>

<v1:TypeID>10</v1:TypeID>

<v1:Subject>Test Ticket created by DN</v1:Subject>

<v1:Description>Ticket Description Value</v1:Description>

<v1:StatusID>101</v1:StatusID>

<!--<v1:AssignTo>NOCTeam</v1:AssignTo> -->

<v1:PriorityID>1</v1:PriorityID>

<v1:TicketNotes>

<v1:NoteDescription>Test Note 1 Value</v1:NoteDescription>

</v1:TicketNotes>

</v1:request>

</v1:CreateTicket>

</soapenv:Body>

</soapenv:Envelope>

Example 2: GetTicketPriorities

This example shows the request and response body of the GetTicketPriorities operation. This operation returns the “TicketPriorityResponse” object.

The TicketPriorityResponse object has two properties:

<Results>: If operation succeeded, API will return the “TicketPriority” object here.
<Error>: If operation failed, API will return an error code and error message here. For the full list of error codes and descriptions, see Public Ticketing API Exceptions.
Request Body

<soapenv:Envelope xmlns:soapenv="http://www.w3.org/2003/05/soap-envelope" xmlns:v1="http://www.continuum.net/its/v1_0">

<soapenv:Header>

<UserID xmlns="TicketService">Testintegrator1</UserID>

<Password xmlns="TicketService">XXXX</Password>

<TokenID xmlns="TicketService">53H48H48H48H49H55H50H56</TokenID>

</soapenv:Header>

<soapenv:Body>

<v1:GetTicketPriorities/>

</soapenv:Body>

</soapenv:Envelope>
